Что нужно реализовать — строго по разделам (оценка за каждый):
1. Авторизация и роли (10%)
- Регистрация/вход с BCrypt
- 4 роли с разными правами: Admin (всё), Manager (документы и товары), Warehouse (инвентаризация), Viewer (только просмотр)
- RBAC — проверять права каждого действия
2. Номенклатура и категории (10%)
- CRUD товаров: название, артикул (уникальный), единица измерения, мин/макс остаток, штрихкод, фото
- Категории — дерево (родитель → потомки), выбрать категорию из TreeView
- Фильтрация: поиск по названию/артикулу/штрихкоду, фильтр по категории
3. Склады и остатки (10%)
- CRUD складов
- Остаток — отдельная сущность StockBalances (товар + склад + количество + резерв + партия + срок годности)
- Свободный остаток = Quantity – ReservedQty
4. Двойные проводки (15%)
- Каждое движение товара = запись в Transactions с Direction '+' или '-'
- Приход: + на склад
- Расход: – со склада
- Перемещение: – со склада А + на склад Б (две проводки в одной транзакции SQL)
- Нельзя провести расход, если свободного остатка меньше чем в документе
- При отмене документа — обратные проводки
5. Документы (15%)
- Приходная накладная — шапка + строки (товар, количество, цена, партия, срок годности)
- Расходная накладная — шапка + строки
- Перемещение — со склада на склад
- У каждого документа уникальный номер (INV-001, TRF-001...)
- Статусы: Draft → Posted → Cancelled
6. Заказы и резервы (15%)
- Заказ клиента: шапка + строки
- Резервирование: при создании заказа резервируем товар (ReservedQty +=)
- Отгрузка: списываем резервы + создаём проводки расхода
- Отмена: Released → ReservedQty –=
- Частичная отгрузка (ShippedQty)
7. Инвентаризация (10%)
- Создание: автоматически подтягиваются ожидаемые остатки со склада
- Ввод фактического количества
- Расхождения = факт – ожидание
- Подтверждение → создаются корректирующие проводки (WriteOff)
8. Отчёты и аналитика (10%)
- ABC-анализ — ранжирование товаров по сумме расхода (A = 80%, B = 15%, C = 5%)
- Оборотно-сальдовая ведомость — остаток на начало, приход, расход, остаток на конец за период
- Дашборд — общее количество товаров, суммарный остаток, кол-во документов
9. Дополнительно (5%)
- Штрихкоды — генерация через ч, поиск товара по вводу со сканера
- PDF — печать накладных через QuestPDF
- Аудит — JSON-логирование всех изменений (кто, когда, что изменил)
Технические требования:
- Стек: Avalonia 11.3 + .NET 10 + EF Core + Dapper + SQL Server LocalDB
- Архитектура: MVVM, DI (Microsoft.Extensions.DependencyInjection)
- Проводки: double-entry (каждая операция = запись в Transactions с Direction)
- Слой read: Dapper (быстрые SQL-запросы остатков, отчётов)
- Слой write: EF Core (создание/обновление сущностей)
- Безопасность: BCrypt для паролей + RBAC для действий
